1. Workshop Objective
By the end of this workshop, participants will be able to:
* Work on real Flutter projects (new development and maintenance)  
* Use professional development environments  
* Convert designs (Figma / XD) into production-ready UI  
* Integrate REST APIs  
* Implement authentication (Google, Apple, Facebook)  
* Handle payment flows (KNET / KeyNet)  
* Manage builds and releases  
* Collaborate using GitHub in a team environment  

2. Weekly Breakdown and Checkpoints

Week 1: Fundamentals & Environment Setup
Objective: Establish a solid technical foundation
Topics
* Flutter overview and architecture  
* Dart essentials (focused on Flutter usage)  
* Project structure and best practices  
* IDE setup (VS Code / Android Studio)  
* Flutter SDK management  
* Emulator and physical device setup  
* Git fundamentals:  
    * Repository creation  
    * Clone, commit, push, pull  
Checkpoint
* Create a Flutter project  
* Initialize a GitHub repository  
* Push the initial commit  
* Run the app successfully on an emulator or real device  


Week 2: UI Development & Design Integration
Objective: Convert designs into a clean Flutter UI
Topics
* Flutter UI fundamentals  
* Layouts, widgets, and responsive design  
* Design handoff from:  
    * Figma  
    * Adobe XD  
* Design systems:  
    * Colors  
    * Typography  
    * Spacing  
* Reusable and scalable UI components  
* Folder and file structuring  
Checkpoint
* Implement 2–3 screens from Figma or XD  
* Apply consistent spacing, fonts, and colors  
* Use reusable widgets  
* Maintain clean project structure  



Week 3: API Integration & State Management
Objective: Build data-driven applications
Topics
* REST API concepts  
* HTTP methods (GET, POST, PUT, DELETE)  
* JSON parsing and data models  
* Error handling strategies  
* Loading and empty states  
* State management introduction (Cubit or Provider)  
* Separation of concerns (UI vs logic)  
Checkpoint
* Integrate a real API  
* Display dynamic data  
* Handle loading, success, and error states  
* Use a proper state management solution  

Week 4: Authentication & User Management
Objective: Implement real-world authentication flows
Topics
* Firebase overview  
* Authentication methods:  
    * Google Sign-In  
    * Facebook Login  
    * Apple Sign-In  
* Token handling  
* Secure local storage  
* Session persistence  
* Logout and session expiration  
Checkpoint
* Implement login screen  
* Enable Google authentication (mandatory)  
* Optional: Apple or Facebook login  
* Maintain user session across app restarts  

Week 5: Payments & Advanced Features
Objective: Exposure to enterprise-level features
Topics
* Payment flow fundamentals  
* KNET / KeyNet overview  
* WebView-based payment integration  
* Handling payment callbacks  
* Environment configuration:  
    * Development  
    * Staging  
    * Production  
* Introduction to deep linking  
* Basic mobile security practices  
Checkpoint
* Implement payment flow (sandbox or mock)  
* Handle success and failure callbacks  
* Secure URLs and parameters  
* Code review and refactoring  

Week 6: Builds, Deployment & Team Workflow
Objective: Prepare developers for production and teamwork
Topics
* Build flavors and environments  
* Exporting builds:  
    * APK  
    * AAB  
    * IPA (overview)  
* Versioning and release management  
* GitHub workflow:  
    * Branching strategy  
    * Pull requests  
    * Code reviews  
* Bug fixing and maintenance  
* Working with legacy projects  
Final Checkpoint
* Complete end-to-end Flutter project including:  
    * UI  
    * API integration  
    * Authentication  
    * Payment flow  
* GitHub pull request with review  
* Final demo and technical discussion  


. Delivery Methodology
* 70% practical work, 30% explanation  
* Each topic follows:  
    * Explanation  
    * Live demonstration  
    * Hands-on task  
* Weekly code reviews  
* Peer reviews  
* Real-world issues and scenarios  

. Final Output for Participants
* Professional GitHub profile  
* Production-ready Flutter project  
* Experience in maintenance and new development  
* Understanding of real team workflows  
* Ready to contribute to enterprise or startup projects
